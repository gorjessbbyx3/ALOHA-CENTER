Aloha Healing Center - Vercel Deployment Information

This page provides information about deploying the Aloha Healing Center application on Vercel.

Note: If you're seeing this page, it means you've accessed a static file on your Vercel deployment. The application should be running at the root URL.

Deployment Requirements

To successfully deploy this application on Vercel, you need:

  1. Database Connection: A PostgreSQL database accessible from Vercel (e.g., Vercel Postgres, Neon, Supabase)
  2. Environment Variables: Proper configuration of all required environment variables
  3. Build Configuration: Correct Vercel build settings

Required Environment Variables

The following environment variables must be set in your Vercel project settings:

DATABASE_URL=postgresql://username:password@host:port/database
NODE_ENV=production
STRIPE_SECRET_KEY=sk_test_your_secret_key
VITE_STRIPE_PUBLIC_KEY=pk_test_your_public_key
CALENDLY_API_KEY=your_calendly_api_key
CALENDLY_USER_URI=https://api.calendly.com/users/your_user_id
CALENDLY_ORG_URI=https://api.calendly.com/organizations/your_org_id

Troubleshooting

Common Issues:
  • Database connection errors - Make sure DATABASE_URL is correctly set and the database is accessible
  • Missing environment variables - Verify all required variables are set in Vercel
  • API connection failures - Check if API routes are correctly configured in vercel.json

Database Migrations

After deploying, you may need to run database migrations. You can do this with the provided script:

node vercel-db-migrate.js
